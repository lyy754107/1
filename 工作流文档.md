# Noodle App — AI 辅助开发工作流

本项目是一个完整的面条点餐 Web App（402×874 移动端单文件应用），通过 **Figma MCP + Claude Code** 实现从设计稿到代码的 AI 辅助开发。

---

## 一、项目概况

### 1.1 最终成果

一个功能完整的移动端点面应用，包含：

- **首页（点面主界面）**：浇头选择、小料选择、辣度/份量/面种类/做法切换，实时碗内可视化
- **入场弹窗**：从屏幕下方滑入，含菜单点餐/自由点餐/端午福利优惠券三个入口，支持手势下滑关闭
- **闪屏广告**：两种路径 — 取消进入首页、自由点餐进入定制动画
- **菜单页**：菜品列表浏览
- **确认订单页**：加餐推荐、价格汇总
- **小恐龙游戏**：Canvas 游戏，等待页 + 游戏中 + 游戏结束三种状态
- **导航栏**：首页/菜单/订单/我的 四个 Tab 切换

### 1.2 技术栈

- 单文件 HTML（CSS + JS 内联），约 5000+ 行
- Node.js 本地服务器（`server.js`，处理 MIME 类型和缓存控制）
- Canvas 2D 游戏引擎
- CSS transition/transform 动画
- Figma MCP 工具链（设计数据读取 + 素材导出）

## 二、AI 辅助开发流程

### 阶段 1：搭骨架 — 从 Figma 到代码

**做了什么**

1. 在 Figma 中设计好首页布局（Frame 402×874），包含碗区域、浇头按钮、底部导航等
2. 用 `figma_get_component_for_development` 读取每个 UI 元素的精确坐标、尺寸、颜色、字体
3. Claude Code 根据 Figma 数据生成 HTML/CSS/JS 骨架

**关键做法**

- 不靠眼睛取色，用 Figma 返回的 hex 值
- 不靠猜测位置，用 Figma 的 `absoluteBoundingBox` 精确坐标
- 数据结构先行：先定义 `toppingDefs`、`seasoningDefs` 等 JS 数据，再渲染 DOM

```javascript
const toppingDefs = [
  {name:'香菇鸡', x:159, y:466, u:'浇头ui/香菇鸡 未选中.png', s:'浇头ui/香菇鸡 选中.png'},
  // ...
];
```

**遇到的困难**

- Figma 实例默认名是 "Property 1=Variant3" 这种无意义字符串，代码无法匹配
- 刚开始不清楚哪些数据从 Figma 取、哪些硬编码

**怎么解决的**

- 在 Figma 中有意义地重命名每个图层（如「香菇鸡」「牛肉」）
- 约定：坐标/颜色/字体从 Figma 取，业务逻辑（价格/规则）手动定义

---

### 阶段 2：素材导出与命名

**做了什么**

1. 从 Figma 导出所有 UI 素材：浇头按钮（选中/未选中两态）、碗底图、面图、小料按钮、导航图标等
2. 建立文件夹结构，按功能分组（`浇头ui/`、`碗/`、`面/`、`导航栏/` 等）

**关键做法 — 文件命名规范**

- **图层名即文件名**：在 Figma 里改好图层名再导出，文件名自然就对了
- **代码要能拼出路径**：
  ```javascript
  const img = `浇头ui/${name} 选中.png`;  // 变量 name 必须和文件名匹配
  ```
- 同组素材命名格式统一：`{名称} {状态}.png`

**遇到的困难**

| 问题 | 原因 | 解决 |
|------|------|------|
| 导出后文件名对不上 | Figma 图层名有全角符号或默认名 | 逐个检查并重命名图层后重新导出 |
| 替换素材后页面没变化 | 浏览器缓存 | `server.js` 加 `Cache-Control: no-cache`，Cmd+Shift+R 硬刷新 |
| 透明区域变白底 | 透明素材用了 JPG | 透明素材统一用 PNG |
| 图片模糊 | 倍率不够 | 导出选 ×2 或 ×3 |

### 阶段 3：交互动效实现

**做了什么**

1. Pill 滑动选择器：浇头类别切换时 pill 平滑滑动
2. 浇头弹出动画：`scale(0)` → `scale(1.25)` → `scale(1)` 弹性缓动
3. 碗内浇头飞入：从碗中心 `translate + scale + rotateX` 到目标位置
4. 3D 翻转：小料按钮 `rotateY(360deg)` + 中途换图
5. 入场弹窗：面板从屏幕下方滑入 + 遮罩淡入 + 标题延迟出现

**关键做法**

- 动效五要素缺一不可：触发 + 动作 + 动画类型 + 时长 + 缓动曲线
- 用 `transform` 而非 `left/top`（GPU 加速）
- 防连点：动画期间设 `el.dataset.animating = '1'`

**遇到的困难**

| 困难 | 原因 | 解决 |
|------|------|------|
| 弹窗滑入动画不触发 | `display:none` → `display:block` 在同一帧内完成，CSS transition 看不到初始状态 | 先设 `display:block` + 强制初始位置 + `offsetHeight` 触发回流 + 再清除内联样式让 transition 生效 |
| 动画速度太快 | 默认 0.4s 不够自然 | 面板调至 0.65s，遮罩 0.5s，标题 0.5s + 0.15s 延迟 |
| Pill 位置偏移 | Figma 标签不等距分布 | 改用绝对定位 + `absoluteBoundingBox` 精确计算 left/width |

**弹窗动画修复的核心代码逻辑：**

```javascript
// 1. 先让元素可见但保持初始状态（屏幕外/透明）
el.style.display = 'block';
mask.style.opacity = '0'; mask.style.transition = 'none';
panel.style.transform = 'translateY(calc(100% + 53px))'; panel.style.transition = 'none';

// 2. 强制回流 — 浏览器注册初始位置
el.offsetHeight;

// 3. 清除内联样式 — CSS transition 从初始状态平滑过渡到 .show 状态
el.classList.add('show');
mask.style.opacity = ''; mask.style.transition = '';
panel.style.transform = ''; panel.style.transition = '';
```

### 阶段 4：图层 z-index 管理

**遇到的最棘手问题：层级混乱**

弹窗、导航栏、闪屏广告三者的显示/隐藏和层级关系非常容易出错：

| 问题 | 表现 |
|------|------|
| 导航栏穿透闪屏广告 | 闪屏广告播放时底部导航栏可见 |
| 弹窗被导航栏遮挡 | 弹窗面板在导航栏下方 |
| 弹窗遮罩盖不住导航栏 | 视觉上导航栏"浮"在弹窗上面 |

**怎么解决的**

1. 明确 z-index 层级表并严格执行：

   ```
   闪屏广告 (200) < 等待页 (300) < 入场弹窗 (350) < 导航栏底 (400) < 导航栏 (401)
   ```

2. 闪屏广告期间导航栏必须 `display:none`，广告结束后再显示：

   ```javascript
   // dismissSplash 的两个分支里都要处理
   document.getElementById('navBar').style.display = '';
   document.querySelector('.nav-bar-bg').classList.add('show');
   showEntryPopup();
   ```

3. `doSwitchTab` 时如果有弹窗先关闭弹窗，避免层级冲突

### 阶段 5：入场弹窗迭代对齐

**做了什么**

1. 从 Figma 导出弹窗相关素材：`弹窗底图.png`、`一起自制小面.png`、`菜单点餐.png`、`自由点餐.png`、`端午福利优惠券.png`
2. 用图片替换 CSS 绘制的 UI
3. 反复调整位置对齐 Figma

**遇到的问题**

| 问题 | 原因 | 解决 |
|------|------|------|
| 按钮位置总是差几个 px | flexbox 布局无法精确匹配 Figma 绝对坐标 | 改为 absolute 定位，每个按钮用 Figma 精确坐标 |
| 菜单点餐高度被拉伸 | flex 容器 `align-items: stretch` 默认行为 | 去掉 flex 容器，每个按钮独立 absolute 定位 |
| 反复微调位置 | 无法直接在浏览器里对比 Figma | 用 Figma 设计数据，直接对应到 CSS left/top/width/height |

**最终采用的 absolute 定位方案：**

```css
.entry-btn-img.btn-menu  { position:absolute; left:20px;  top:54px;  width:173px; height:159px; }
.entry-btn-img.btn-custom { position:absolute; left:211px; top:55px;  width:170px; height:160px; }
.entry-coupon-img         { position:absolute; left:20px;  top:228px; width:359px; height:132px; }
```

### 阶段 6：手势交互

**做了什么**

1. 弹窗面板支持下滑关闭手势
2. 滑动超过 100px 触发关闭，否则回弹

**遇到的困难**

- 页面使用了 CSS `zoom` 缩放，`touchmove` 的 `clientY` 是物理像素，需要除以 zoom scale 才能对应 CSS 坐标

**怎么解决的**

```javascript
var scale = parseFloat(getComputedStyle(document.querySelector('.app')).zoom) || 1;
panel.style.transform = 'translateY(' + (_entrySwipeDY / scale) + 'px)';
```

### 阶段 7：音效集成

**做了什么**

1. 从视频提取音频片段，转换为 WAV 格式
2. 按钮点击时播放气泡弹出音效（`pop.wav`，约 0.7s）

**遇到的困难**

| 问题 | 原因 | 解决 |
|------|------|------|
| 移动端音效不响 | `Audio.play()` 在 `setTimeout` 回调中丢失了用户操作上下文 | 音频播放必须在 click/touch 的同步代码中调用 |
| M4A 在 iOS 上偶尔不播放 | iOS Safari 对 M4A 支持不稳定 | 统一用 WAV PCM 格式 |
| 中文文件名导致 ffmpeg 报错 | 终端编码问题 | 先复制到 `/tmp/` 再处理 |

---

### 阶段 8：游戏模块

**做了什么**

在等待页集成了一个小恐龙跑酷游戏（Canvas 2D），包含：
- 等待页面（倒计时 + 小恐龙动画）
- 游戏中（跳跃 + 障碍物碰撞）
- 游戏结束页面
- 底图切换（不同状态用不同 Figma 背景图）

**遇到的困难**

- 游戏 Canvas 尺寸与设计尺寸有偏移（412px vs 402px）
- 旧 GIF 素材有残影（帧叠加）
- 游戏状态切换时 Figma 背景图与游戏元素图层重叠

**怎么解决的**

- Canvas 宽度修正为 402px，`left:0` 对齐 app frame
- GIF 重新编码：`disposal=2` + 白底背景，消除帧间残影
- 游戏元素用 GPU 加速层（`transform: translateZ(0)`）

## 三、核心经验总结

### 3.1 AI 辅助开发的成功模式

1. **设计先行**：Figma 里把设计做完整，AI 才能准确读取和还原
2. **图层命名决定代码质量**：Figma 图层名 = 文件名 = 代码中的变量值，命名不一致会导致大量返工
3. **精确数据优于主观判断**：坐标用 `absoluteBoundingBox`，颜色用 hex 值，不要靠眼睛
4. **用 absolute 定位对齐 Figma**：flexbox/grid 在像素级还原时不如 absolute 可控
5. **动画要理解浏览器渲染机制**：`display:none` 跳过 transition、`offsetHeight` 强制回流、`transform` 比 `left/top` 更高效

### 3.2 最容易翻车的点

| 排名 | 问题 | 预防 |
|------|------|------|
| 1 | 图层 z-index 混乱 | 先画层级表，每个元素标注 z-index |
| 2 | 素材缓存导致调试误判 | 服务器加 `Cache-Control: no-cache` |
| 3 | CSS transition 不触发 | 检查 `display:none`、初始状态是否被注册 |
| 4 | 手机端音效不响 | 音频播放必须在用户操作同步代码中 |
| 5 | inline style 覆盖 CSS class | 尽量用 class 控制状态，少用 `el.style.xxx` |

### 3.3 迭代节奏

- 每次只改一个关注点，改完立刻在浏览器验证
- Figma 数据是锚点 — 不确定时回到 Figma 重新读数据
- 素材更新后必须硬刷新（Cmd+Shift+R）

## 四、技术备忘

### 4.1 项目参数

| 参数 | 值 |
|------|-----|
| 设计尺寸 | 402 × 874 px |
| 碗区域 | (107, 242)，192 × 192 px |
| 面中心 | (96, 96) 在 192×192 碗内 |
| 浇头距中心半径 | 80px（10 个点位均匀分布） |
| CSS 缩放 | `--as` 自定义属性控制 zoom |

### 4.2 定价规则

- 素面 ¥8，第一个浇头按套餐价，额外浇头按单独价
- 份量：中份 +0 / 大份 +3 / 巨大 +6
- 小料煎蛋 ¥2，其余免费

### 4.3 z-index 层级表

```
游戏 Canvas (1) < 页面内容 (10-11) < 闪屏广告 (200) < 等待页 (300) < 入场弹窗 (350) < 导航栏底 (400) < 导航栏 (401) < Loading (9999)
```

### 4.4 关键文件

| 文件 | 作用 |
|------|------|
| `index.html` | 单文件应用，全部 HTML/CSS/JS |
| `server.js` | Node 服务器，MIME 类型 + Cache-Control |
| `server.log` | 服务器运行日志 |

### 4.5 常用命令

```bash
# 启动本地服务器
node server.js

# 外网访问（手机测试）
npx localtunnel --port 8081

# 从视频提取 WAV 音频
afconvert -f WAVE -d LEI16@44100 input.mp4 output.wav
```
